
토탈커맨드 폴더바 1.7

제에게 있어 토탈커맨드는 PC켜짐과 동시에 실행되어 PC끌때 종료되는 몇 안되는 프로그램입니다.
이는 다소 개인적인 프로그램이며, 저에게 있어서 토탈커맨드의 유일한 단점인
툴바에 글자를 넣을 수 없는 문제를 어떻게든 해결하려다보니 만드어졌습니다.
이 프로그램의 주 기능은 토탈커맨드에서 폴더 전환을 빠르게 하기 입니다.
토탈커맨드를 사용하시는 사용자만 다운로드하십시요. 

혹시나 필요할 지 모르는 사용자들이 있어서 공개합니다.
라이센스는 물론 프리웨어이며, 무차별 복제가 가능합니다. 출처같은거 안 남겨도 되구요,
사실상 거의 마지막 버전입니다.

1.7
수정일: 06-07-31
수정: 서브폴더를 전환하게 하는 팝업메뉴의 순서를 변경하고 , 삭제하거나 , 메뉴제목을 수정할 수 있습니다.

[메뉴얼]
1. Ctrl+드래그를 하면 바를 움직이게 합니다.
2. 폴더를 추가하려면 폴더나 파일을 폴더바로 드래그 앤 드랍하면 됩니다.
3. 폴더를 [아래로 추가]한 경우에는 폴더 아이콘의 색이 붉은색으로 나오면서, 툴바의 버튼위에서 
   오른쪽 마우스 버튼을 눌러서 해당 서브 메뉴를 출력합니다.
4. 서브메뉴가 있는 툴 버튼 위에서 Ctrl+오른쪽 클릭을 하게 되면 버튼을 수정할 수 있는 메뉴가 나옵니다.
4. 팝업메뉴를 출력시에 Ctrl+클릭하면 해당 항목을 삭제합니다.
5. 트레이에 아이콘을 출력시키지 않으려면 아래처럼 설정하십시요.
    --TCmdbar.ini--
    [main]
    bTray=0

1.3 log
06-01-01
김대정이 작성
ㅇ 아이콘 변경, 그냥 취향의 문제임
ㅇ 서브 폴더를 출력하는 방식을 마우스 오른쪽 버튼으로 교체, 서브폴더가 있는 버튼에서는 Ctrl+우클릭하게 되면 기존의 설정 메뉴가 나오고 그냥 우클릭시에는 서브폴더 메뉴가 나옮. 이유: 기존방식에서는 아이콘의 위치로 마우스를 정확히 위치시키는데 의외로 주의를 기울려야하는 부담감이 있음. 
ㅇ 서브폴더 메뉴에서 Ctrl+클릭으로 해당 항목 삭제 기능에서 일부 삭제되지 않는 버그를 제거
ㅇ 버튼 삭제시 관계된 서브폴더를 삭제하지 않아서 같은 폴더를 바의 뒤에 새로추가시에 자동으로 서브폴더가 있다고 나오는 버그 제거
ㅇ 버튼의 경로를 수정시에 서브폴더가 사라지는 버그 제거(이 문제로 인해 서브폴더를 1.3버전에서는 처음에는 다시 설정하여야 함)
^^ 바가 토탈커맨드를 따라 다니는 기능은 추후에 업그레이드 시키겠음

문서버전 : 1.0
문서작성일 : 2005-04-21
작성자 : 김대정
수정사항 : 최초작성

ㅇ 이 프로그램은 토탈커맨드의 빠른 폴더 전환 기능을 목적으로 제작되었습니다.
   토커제작자에게 기존의 버튼위에 텍스트를 삽입할 수 있도록 강력히 건의했거만 불가능하다고 하여 할 수 없이
   토커의 부족한 2%를 채우기로 결정하고 몇시간 뚝딱하여 만들었습니다. 

ㅇ 토탈커맨드가 활성화되면 바도 활성화가 됩니다. 

ㅇ 버튼 정보의 수정창에서 USER ID 란에는 701부터 숫자로 지정하면 토커의 시작메뉴의 폴더전환 메뉴가 동작합니다. 토커의 시작메뉴의 구성을 참조
   비워두고 폴더패스에 폴더의 경로를 지정하면 폴더경로로 직접 이동합니다.
